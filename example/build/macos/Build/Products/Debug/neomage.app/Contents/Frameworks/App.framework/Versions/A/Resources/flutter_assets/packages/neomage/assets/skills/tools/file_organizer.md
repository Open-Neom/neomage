# File Organizer

Intelligently organize files and folders by analyzing content, finding duplicates, and suggesting better structures.

- Start by understanding scope: which directory, main problem, files to avoid, aggressiveness level
- Analyze current state: total files, type breakdown, size distribution, date ranges, obvious issues
- Organize by type (documents, images, code, archives), purpose (work/personal, active/archive), or date
- Find duplicates using file hashes for exact matches and filename similarity for near-duplicates
- Always present a complete organization plan and get user approval before making any changes
- Always confirm before deleting anything; log all moves for potential undo
- Folder naming: clear, descriptive, no spaces (use hyphens), specific names over generic
- File naming: include dates (YYYY-MM-DD), be descriptive, remove download artifacts like "(1)"
- Archive projects not touched in 6+ months; archive before deleting when uncertain
- Provide maintenance tips: weekly sort downloads, monthly archive completed projects, quarterly dedup
