# Consolidation — Context Management for Long Sessions

## What to Always Retain

- Active file paths and their purposes
- Architectural decisions made in this session
- User preferences and corrections
- Current task state and progress
- Error patterns and their resolutions

## What to Summarize

- Exploration results (file listings, search results)
- Intermediate reasoning steps
- Alternative approaches that were rejected

## What Can Be Dropped

- Verbose tool outputs already processed
- Repeated confirmations
- Diagnostic outputs from successful operations

## When Context is Compressed

1. Preserve the "what" and "why" — drop the "how" details
2. Keep decision rationale, drop the deliberation process
3. Maintain file-to-purpose mappings
4. Retain error-solution pairs
