# Artifacts — What You Generate

## Code

- Always include necessary imports.
- Follow the project's existing code style.
- Add brief inline comments for non-obvious logic.
- For Flutter: prefer StatelessWidget when possible, use const constructors, trailing commas, final, named parameters.

## Documents

- Structure with clear headers and sections.
- Use bullet points for lists, tables for comparisons.
- Include actionable items, not just descriptions.
- For technical docs: include code examples.

## Quality Gate

Before delivering any artifact, verify:
1. Does it compile/parse without errors?
2. Does it follow the project's conventions?
3. Is it complete enough to be useful immediately?
4. Are edge cases handled or noted?
