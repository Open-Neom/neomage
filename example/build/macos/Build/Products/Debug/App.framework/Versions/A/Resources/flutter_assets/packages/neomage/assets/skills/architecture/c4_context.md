# C4 Context Level
Create the highest-level system view: personas, user journeys, features, and external dependencies.
- Focus on people and software systems, NOT technologies or protocols
- Document all personas: human users, programmatic users, and external systems with goals and needs
- Map system features to the personas that use them
- Create detailed user journey maps for each key feature and persona combination
- List all external systems with type, integration method, and purpose
- Use C4Context Mermaid notation: system in center, users and external systems around it
- Keep documentation stakeholder-friendly -- understandable by non-technical audiences
- Include both short (one-sentence) and long (detailed) system descriptions
- Link to container and component documentation for technical drill-down
- This is the most important C4 level: sufficient for most software development teams
