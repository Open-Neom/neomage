# Code Review Excellence

Transform code reviews from gatekeeping into knowledge sharing through constructive feedback and systematic analysis.

- Read context, requirements, and test signals before reviewing any code
- Review for correctness, security, performance, and maintainability in that order
- Provide actionable feedback grouped by severity: blocking, important, minor
- Ask clarifying questions when intent is unclear rather than assuming
- Balance thoroughness with practical development velocity
- Use reviews as teaching opportunities: explain the "why" behind suggestions
- Focus on patterns and design, not just line-level issues
