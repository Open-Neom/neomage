# Incident Response

Multi-phase incident management following modern SRE principles for rapid resolution and organizational learning.

- Phase 1 (Detection): classify severity, analyze observability data, implement immediate mitigation
- Phase 2 (Investigation): deep system debugging, security assessment, performance analysis, Five Whys
- Phase 3 (Resolution): implement minimal viable fix, staged rollout, validate with health checks
- Phase 4 (Communication): status page updates, internal engineering details, executive summaries, support briefings
- Phase 5 (Postmortem): blameless analysis, complete timeline, action items with owners, monitoring enhancements
- Incident command structure: commander (decisions), tech lead (investigation), comms lead (stakeholders)
- Success: restore within SLA, classify within 5 minutes, communicate every 15-30 minutes, postmortem within 48 hours
