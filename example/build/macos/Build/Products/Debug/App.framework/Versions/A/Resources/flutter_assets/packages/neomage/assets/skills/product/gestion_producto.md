# Gestion de Producto

- El product manager es el nexo entre negocio, tecnologia y usuario; prioriza valor sobre funcionalidades
- Discovery antes de delivery: validar que el problema existe antes de construir la solucion
- Framework RICE para priorizacion: Reach x Impact x Confidence / Effort
- OKRs conectan vision de producto con metricas medibles: Objective (cualitativo) + Key Results (cuantitativos)
- Roadmap comunica direccion, no compromisos de fechas; dividir en Now/Next/Later

## Ciclo de producto
1. Descubrimiento: entrevistas, datos de uso, analisis competitivo
2. Definicion: user stories con criterios de aceptacion, prototipo/mockup
3. Desarrollo: sprint con scope controlado, demos frecuentes
4. Lanzamiento: feature flags, rollout gradual, metricas de exito definidas
5. Evaluacion: medir impacto vs hipotesis, iterar o pivotar

## Metricas de producto
- Activacion: % usuarios que alcanzan el "aha moment"
- Retencion: cohortes semanales/mensuales, churn rate
- Engagement: DAU/MAU ratio, session duration, feature adoption
- Monetizacion: ARPU, conversion free→paid, expansion revenue

## User stories efectivas
- Formato: Como [persona], quiero [accion], para [beneficio]
- Criterios de aceptacion: Given/When/Then, edge cases explicitos
- Slice vertical: cada story entrega valor end-to-end

## Anti-patrones
- Construir features porque un cliente grande lo pidio (sin validar mercado)
- Roadmap como lista de features sin vision ni hipotesis
- Medir output (features lanzadas) en vez de outcome (impacto en metricas)
