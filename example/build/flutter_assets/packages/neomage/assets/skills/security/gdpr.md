# GDPR Data Handling

Practical GDPR compliance: consent management, data subject rights, and privacy-by-design architecture.

- Legal basis: identify lawful basis for each processing activity (consent, contract, legitimate interest, legal obligation)
- Consent: granular, freely given, informed, withdrawable; store proof of consent with timestamp and scope
- Data subject rights: implement access, rectification, erasure, portability, restriction, and objection workflows
- Data minimization: collect only what is necessary, enforce retention periods, auto-delete expired data
- Privacy by design: encrypt at rest and in transit, pseudonymize where possible, default to most private settings
- Data Processing Agreements: required for all third-party processors; define purpose, duration, and security measures
- Breach notification: detect within 72 hours, assess risk, notify supervisory authority and affected individuals if high risk
- Records of processing: maintain an up-to-date register of all processing activities with purposes and categories
- Cross-border transfers: use Standard Contractual Clauses or adequacy decisions for data leaving the EU
- DPIA: conduct Data Protection Impact Assessments for high-risk processing before deployment
