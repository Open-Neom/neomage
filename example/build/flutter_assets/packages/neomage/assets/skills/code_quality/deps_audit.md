# Dependency Audit

Analyze project dependencies for vulnerabilities, license compliance, and supply chain security risks.

- Inventory both direct and transitive dependencies from manifest files
- Run vulnerability scans and cross-reference against CVE databases
- Check license compliance: identify incompatible or risky licenses
- Prioritize fixes by severity and exposure surface
- Propose upgrade paths with compatibility notes and breaking change warnings
- Verify upgrades in staging before production rollout
- Never publish sensitive vulnerability details to public channels
