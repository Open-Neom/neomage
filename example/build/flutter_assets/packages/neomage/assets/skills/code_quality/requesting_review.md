# Requesting Code Review

Prepare and request code reviews at the right time to catch issues before they compound.

- Mandatory review points: after each task, after major features, before merge to main
- Provide clear context: what was implemented, requirements/plan it fulfills, base and head SHAs, brief description
- Act on feedback by priority: fix critical immediately, fix important before proceeding, note minor for later
- Push back on incorrect feedback with technical reasoning and evidence (tests, working code)
- Never skip review because "it's simple" -- simple changes still benefit from fresh eyes
- Integrate with workflows: review after each task in iterative development, after every 3-task batch in plan execution
- Catch issues early to prevent compounding technical debt across subsequent work
