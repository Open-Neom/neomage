# C4 Code Level
Document individual code directories with complete function signatures, classes, dependencies, and relationships.
- For each directory: name, description, location, language, and purpose
- Document all functions with full signatures: parameters, types, return types, and dependencies
- Document all classes with methods, attributes, and relationships
- List internal dependencies (other code in repo) and external dependencies (libraries, services)
- Use classDiagram for OOP code showing inheritance, composition, and interfaces
- Use flowchart for functional code showing data pipelines and function composition
- Use classDiagram with <<module>> stereotype for functional modules with exports
- Include source code file paths and line numbers for traceability
- Only create code diagrams for complex components; most teams find higher levels sufficient
- This is the lowest C4 level: it feeds into component-level synthesis
