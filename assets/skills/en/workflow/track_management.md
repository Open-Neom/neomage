# Track Management

Create, manage, and complete Conductor tracks -- the logical work units organizing features, bugs, and refactors.

- Tracks flow through three phases: specification (spec.md), planning (plan.md), and implementation
- spec.md defines what needs to be built and why; plan.md breaks it into actionable tasks
- Each track has a lifecycle: created, specified, planned, in-progress, completed
- Use status markers in track files to communicate progress clearly
- Register all tracks in the central tracks.md registry for visibility
- Track types: feature (new capability), bug (defect fix), refactor (structural improvement)
- Keep specs focused on requirements and acceptance criteria, not implementation details
- Plans should decompose work into independently testable and committable tasks
