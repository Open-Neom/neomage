# README Efectivo

- El README es la puerta de entrada al proyecto; determina si alguien lo usa o lo abandona en 30 segundos
- Estructura minima: titulo, descripcion (1 parrafo), instalacion, uso rapido, contribucion, licencia
- La descripcion responde: que hace, para quien, por que es diferente (elevator pitch de 2 oraciones)
- Quick Start debe llevar al usuario de cero a resultado funcional en menos de 5 minutos
- Badges comunican salud: CI status, cobertura, version, licencia, ultima actividad

## Secciones por madurez del proyecto
- **MVP**: titulo, descripcion, instalacion, uso, licencia
- **Crecimiento**: + contribucion, changelog, roadmap, FAQ
- **Maduro**: + arquitectura, guia de migracion, sponsors, community

## Principios
- Asumir que el lector no tiene contexto previo del proyecto
- Comandos ejecutables copy-paste (no pseudocodigo)
- Screenshots o GIFs para proyectos visuales
- Links a documentacion extensa en vez de duplicar en README
- Mantener actualizado: README desactualizado es peor que no tener README

## Anti-patrones
- README vacio o generado automaticamente sin personalizar
- Instrucciones de instalacion que no funcionan
- Historia del proyecto antes del valor para el usuario
- Links rotos a documentacion o demos
