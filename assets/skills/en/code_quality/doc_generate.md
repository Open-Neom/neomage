# Documentation Generation

Generate comprehensive, maintainable documentation from code using automated analysis and best practices.

- Identify required doc types (API, architecture, user guides) and target audiences
- Extract information from code, configs, comments, and type signatures
- Generate docs with consistent terminology, structure, and cross-references
- Never expose secrets, internal URLs, or sensitive data in documentation
- Keep documentation synchronized with code as a living artifact
- Include automation hooks (linting, CI checks) to validate accuracy over time
- Document gaps and assumptions alongside the generated output
