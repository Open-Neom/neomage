# PR Enhancement

Create high-quality pull requests that facilitate efficient code reviews through clear descriptions and proper documentation.

- Analyze the diff to identify intent, scope, and affected components
- Summarize what changed and why, not just listing files
- Highlight breaking changes, migration steps, and rollout considerations
- Include test results and coverage impact
- Add reviewer guidance: what to focus on, risks, and rollback notes
- Keep PRs small and focused; split large changes into reviewable chunks
- Use checklists to ensure nothing is missed before requesting review
