# Code Explanation

Transform complex code into clear explanations through narratives, diagrams, and step-by-step breakdowns.

- Start with a high-level summary of purpose and flow before drilling into components
- Assess structure, dependencies, and complexity hotspots first
- Use diagrams, pseudocode, or annotated snippets when they aid understanding
- Explain progressively: overview first, then key parts, then edge cases
- Call out pitfalls, implicit assumptions, and key terminology
- Tailor depth to the audience: onboarding walkthroughs vs. deep-dive analysis
- End with suggested next steps and areas that need further exploration
