# Compliance Check

Regulatory compliance auditing for GDPR, HIPAA, SOC2, PCI-DSS, and other standards.

- Start with gap analysis: compare current controls against target framework requirements, rate each gap by severity
- GDPR: verify lawful processing basis, consent mechanisms, DSR workflows, DPIAs, breach notification procedures
- HIPAA: audit PHI handling, access controls, audit logs, BAA agreements, encryption at rest and in transit
- SOC2: assess Trust Service Criteria (security, availability, processing integrity, confidentiality, privacy)
- PCI-DSS: verify cardholder data isolation, network segmentation, encryption, access controls, monitoring
- Automated monitoring: implement continuous compliance checks, policy-as-code with OPA, automated evidence collection
- Audit trails: maintain tamper-evident logs of access, changes, and security events with adequate retention
- Technical controls: encrypt sensitive data, enforce MFA, implement RBAC, maintain patch management
- Documentation: maintain privacy policies, consent forms, data processing records, and incident response plans
- Never claim formal compliance without a certified audit; use assessments to identify and prioritize remediation
