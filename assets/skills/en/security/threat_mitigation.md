# Threat Mitigation Mapping

Map identified threats to security controls for effective defense-in-depth planning.

- For each threat, identify one or more security controls that reduce likelihood or impact
- Control categories: preventive (block attacks), detective (identify attacks), corrective (recover from attacks)
- Defense-in-depth: layer multiple control types; no single control should be the sole defense for a critical threat
- Priority matrix: rank mitigations by cost, implementation effort, risk reduction, and coverage breadth
- Coverage analysis: ensure every high/critical threat has at least one implemented control; flag gaps
- Control effectiveness: validate that controls actually work via testing (penetration tests, red team exercises)
- Residual risk: after controls are applied, document remaining risk and get explicit acceptance from stakeholders
- Remediation roadmap: phase implementation by priority -- quick wins first, then strategic controls
- Traceability: maintain links from threat to control to implementation to test evidence
- Review cadence: reassess mappings quarterly, after incidents, after architecture changes, or new threat intelligence
