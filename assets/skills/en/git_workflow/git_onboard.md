# Git Onboarding

Structured onboarding plan for new team members covering technical setup, codebase immersion, and team integration.

- Pre-boarding: provision accounts (GitHub, Slack, AWS), ship hardware, assign onboarding buddy
- Day 1: manager welcome, company culture overview, team intros, IT setup, security training
- Week 1: architecture overview, dev environment setup, branching strategy, first "good first issue" PR
- Development setup: IDE, linters, formatters, Git config, SSH keys, Docker, testing frameworks
- Team integration: daily buddy check-ins first week, shadow standups and retros, 1:1s with each team member
- 30-day milestone: 10+ commits merged, all training complete, first bug fix in production
- 60-day milestone: own a small feature end-to-end, shadow on-call, contribute to design discussions
- 90-day milestone: independent feature delivery, active code reviews, mentor newer members
- Track metrics: time to first commit, time to first production deploy, ramp-up velocity
