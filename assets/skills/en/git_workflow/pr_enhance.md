# PR Enhancement

Optimize pull requests for efficient code review with comprehensive descriptions and proper documentation.

- PR summary: include executive summary with key metrics (files changed, lines added/removed)
- Detailed description: explain what changed, why, and any context not obvious from the code
- Review checklist: generate context-aware review items specific to the changes
- Risk assessment: identify risky areas with mitigation strategies for reviewers to focus on
- Test coverage: show before/after coverage comparison highlighting gaps
- Size recommendations: suggest splitting large PRs into smaller, reviewable units
- Include visual aids (diagrams, screenshots) for UI changes or architectural modifications
- Goal: create PRs that are a pleasure to review with all necessary context upfront
